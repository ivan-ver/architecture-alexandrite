## Cуществующие и потенциальные проблемные места:

1) Долгий расчёт стоимости в MES (до 30 минут) (плохой UX, возможные таймауты).
2) Ручное тестирование QA (задержки релизов, до месяца).
3) Синхронное взаимодействие онлайн-магазина и MES (магазин блокируется при расчёте цены).
4) Нет staging-окружения (риск багов в prod).
5) Монолитная БД (один инстанс на запись/чтение) (риск перегрузки).
6) Недостаток автоматизации CI/CD → (медленные и рискованные деплои).
7) Нехватка специалистов (1 DevOps, нет SRE, слабый React).
8) Нет кеширования расчётов (повторные запросы считаются заново).
9) Нет отказоустойчивости MES (если упадёт, заказы не будут считаться).
10) Нет мониторинга (сложно выявлять узкие места).
11) Проблемы с масштабированием (при росте нагрузки система может не справиться).

## Инициативы для устранения проблем:

| Инициатива                        | Описание                                                                |
|-----------------------------------|-------------------------------------------------------------------------|
| Асинхронный расчёт стоимости      | 	MES возвращает ID задачи, результат приходит через WebSocket/callback. |
| Автоматизация E2E-тестов          | 	Внедрение Playwright/Cypress для ускорения проверок.                   |
| Введение staging-окружения        | 	Добавить промежуточную среду перед prod.                               |
| Репликация БД                     | 	Чтение с реплик, запись в мастер (PostgreSQL/Yandex Managed DB).       |
| Кеширование расчётов (Redis)      | 	Сохранять результаты, чтобы избежать повторных вычислений.             |
| Event-Driven Architecture (EDA)   | 	MES, CRM и магазин общаются через брокер (Kafka/RabbitMQ).             |
| API Gateway                       | 	Единая точка входа для API (Kong/Yandex API Gateway).                  |
| Найм React-разработчика           | 	Усиление фронтенда MES.                                                |
| Второй DevOps/SRE                 | Для мониторинга и отказоустойчивости.                                   |
| Canary-деплой                     | 	Постепенный rollout новой версии в prod.                               |
| Оптимизация алгоритмов MES        | 	Параллельные вычисления, GPU-ускорение.                                |
| Мониторинг (Prometheus + Grafana) | 	Контроль загрузки системы, быстрое выявление проблем.                  |

### Критерии приоритизации:
1) Влияние на бизнес (UX, скорость обработки заказов).
2) Риски (насколько проблема критична для стабильности).
3) Сложность реализации (время, ресурсы).

## Целевая архитектура через полгода:
### Слой доступа и балансировки
* Cloudflare/WAF - защита от DDoS, кеширование статики
* API Gateway:
  - Единая точка входа для всех API
  - Rate limiting, аутентификация, кеширование ответов
  - Маршрутизация запросов к соответствующим сервисам

### Сервисный слой
* Online Store Service
  * Назначение: работа с клиентами, корзина, заказы
  * Технологии: Vue 3 + Spring Boot 3
  * Особенности:
    - Статический контент через CDN
    - WebSocket для реального времени
* CRM Service
  * Назначение: управление клиентами, заказами, продажами
  * Технологии: Vue 3 + Spring Boot 3
  * Особенности:
    - Интеграция с email/SMS сервисами
    - Dashboard для аналитики
* MES Service
  * Назначение: расчёт стоимости, управление производством
  * Технологии: React + .NET 7
  * Особенности:
    - оновые workers для расчётов
    - PU-ускорение для сложных моделей

### Асинхронная коммуникация
* Kafka Cluster (вместо RabbitMQ)
* Schema Registry - валидация схем сообщений
* Kafka Connect - интеграция с БД

### Слой данных
* Кеширование (Redis Cluster)
  * Кеш результатов расчётов стоимости
  * Сессии пользователей
  * Очереди для фоновых задач
* Основная БД (PostgreSQL HA)
  * Архитектура: 1 мастер + 2 реплики для чтения
  * Шардинг: по клиентам для масштабирования
  Бэкапы: автоматические в S3
* Хранилище файлов (Yandex Object Storage)
  * 3D-модели клиентов
  * Логи производственного процесса
  * Резервные копии

### Мониторинг и логирование
* метрики
* логи и трассировка
